## 2024-05-24 - String Allocations Inside Leptos Render View Loops
**Learning:** Leptos updates node properties frequently on events (like scrolling). String allocations inside high-frequency reactive scopes and rendering components (such as within class conditionals or key derivations in virtual lists) dramatically negatively impact rendering performance. Avoid returning `String` in dynamic properties. Returning `&'static str` saves large reallocation overheads, and removing string components like `item.name` from loop node `key` fields saves expensive cloning while patching DOM layouts.
**Action:** When mapping over iterables in `<For>` components, verify that the `key` prop returns lightweight IDs and skips strings or cloning properties. When returning strings dynamically in reactive nodes, return a static reference `&'static str` instead of allocating a new string `String` via `to_string()`.
